# 语音系统用户手册

## 功能概述

来福的语音系统包括三个部分：

- **语音识别 (STT)**：听懂你说的话，基于 Whisper 本地离线识别
- **语音合成 (TTS)**：来福说话的声音，支持 Edge TTS 在线语音和 GPT-SoVITS 声音克隆
- **语音活动检测 (VAD)**：自动检测你什么时候开始说话、什么时候停下来，无需手动按按钮

---

## 语音识别 (STT)

### 工作方式

来福使用 faster-whisper 进行本地语音识别，完全离线运行，无需联网。

### 模型选择

| 模型 | 大小 | 速度 | 准确度 | 适用场景 |
|------|------|------|--------|----------|
| tiny | 39MB | 最快 | 一般 | 配置较低的电脑 |
| base | 74MB | 快 | 较好 | 日常使用 |
| **small** | 244MB | 中等 | **推荐** | 中文识别效果最佳 |
| medium | 769MB | 较慢 | 很好 | 高精度需求 |
| large | 1.5GB | 慢 | 最好 | 多语言混合 |

### 配置方法

在 `python-service/config.json` 中修改 `stt` 部分：

```json
{
  "stt": {
    "provider": "whisper",
    "model": "small"
  }
}
```

### 使用本地模型（推荐）

下载 faster-whisper 模型到本地可以避免首次启动时的下载等待：

1. 从 [HuggingFace Mirror](https://hf-mirror.com) 下载对应模型
2. 放到本地目录（如 `E:\LLM\backbone\Voice\faster-whisper-small`）
3. 服务启动时会自动检测并使用本地模型

### 设备自动检测

- **有 NVIDIA GPU**：自动使用 CUDA + float16 加速
- **仅 CPU**：自动使用 int8 量化，速度稍慢但仍可用

## 语音合成 (TTS)

### 路由优先级

来福的语音合成使用智能路由，按优先级依次尝试：

```
1. 预合成音频片段 (Clips)     ← 速度最快，适合固定台词
      ↓ 没有匹配
2. 磁盘缓存 (Cache)           ← 之前合成过的相同文本
      ↓ 缓存未命中
3. GPT-SoVITS 推理            ← 来福克隆声音（如果已配置）
      ↓ 推理失败或未就绪
4. Edge TTS 兜底              ← 微软在线语音（需联网）
```

### Edge TTS 配置

Edge TTS 是默认的在线语音引擎，无需额外安装。

在 `config.json` 中配置：

```json
{
  "tts": {
    "provider": "edge",
    "voice": "zh-CN-XiaoxiaoNeural"
  }
}
```

#### 可用中文语音

| 短名 | 完整名称 | 特点 |
|------|----------|------|
| xiaoxiao | zh-CN-XiaoxiaoNeural | 女声，活泼（默认） |
| xiaoyi | zh-CN-XiaoyiNeural | 女声，温柔 |
| yunxi | zh-CN-YunxiNeural | 男声，活泼 |
| yunjian | zh-CN-YunjianNeural | 男声，稳重 |
| xiaochen | zh-CN-XiaochenNeural | 女声，成熟 |
| xiaohan | zh-CN-XiaohanNeural | 女声，甜美 |

### GPT-SoVITS 声音克隆

让来福用你自定义的声音说话！

#### 前置条件

1. 安装 GPT-SoVITS（放在 `tools/GPT-SoVITS/` 目录下）
2. 训练好模型并导出权重文件

#### 语音包目录结构

```
python-service/data/voices/laifu-clone/
├── config.json              # 语音包配置
├── gpt_weights.pth          # GPT 模型权重
├── sovits_weights.pth       # SoVITS 模型权重
└── reference.wav            # 参考音频（3-10 秒）
```

#### config.json 格式

```json
{
  "models": {
    "gpt": "gpt_weights.pth",
    "sovits": "sovits_weights.pth"
  },
  "reference": {
    "audio": "reference.wav",
    "text": "参考音频对应的文本内容",
    "lang": "zh"
  },
  "inference": {
    "top_k": 20,
    "top_p": 0.85,
    "temperature": 0.6,
    "speed": 1.0
  }
}
```

#### 推理参数说明

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| top_k | 20 | 1-100 | Token 选择范围，越大越多样 |
| top_p | 0.85 | 0-1 | 核心采样概率 |
| temperature | 0.6 | 0-1 | 生成多样性，越高越随机 |
| speed | 1.0 | 0.5-2.0 | 语速倍数 |

#### 启动行为

- 服务启动时自动检测 `data/voices/laifu-clone/` 目录
- 如果存在有效的 config.json，自动注册并激活为当前语音
- GPT-SoVITS 模型在后台线程预热加载，期间使用 Edge TTS 兜底
- 预热完成后自动切换到克隆声音

### 语音包管理

通过系统托盘右键 → 语音管理，可以：

- 查看所有已安装的语音包
- 切换当前激活的语音
- 试听语音效果
- 新建/删除语音包

#### API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/tts/voices` | GET | 列出所有语音包 |
| `/api/tts/voices/{id}/activate` | POST | 激活指定语音包 |
| `/api/tts/voices/{id}` | DELETE | 删除语音包 |

### 预合成音频片段

可以为来福预录固定音效（叫声、情绪声等），放在语音包的 clips 目录中。

在语音包的 `config.json` 中配置：

```json
{
  "clips": {
    "barks": {
      "short": "clips/bark_short.wav",
      "long": ["clips/bark_1.wav", "clips/bark_2.wav"]
    },
    "emotions": {
      "happy": "clips/emotion_happy.wav"
    }
  }
}
```

- 支持单个文件或多个文件（随机选一个播放）
- 预合成片段优先级最高，无需等待推理

## 语音活动检测 (VAD)

### 工作方式

来福使用 Web Audio API 进行能量检测，无需任何外部依赖：

1. 常驻后台监听麦克风
2. 自动检测你什么时候开始说话
3. 你停顿后自动结束录音并发送识别
4. 自适应环境噪声基线

### 核心参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| energyThreshold | 18 | RMS 能量阈值（0-255），高于此值认为在说话 |
| silenceDuration | 900ms | 静音持续多久触发结束 |
| minSpeechMs | 300ms | 说话低于此时长认为是误触发 |
| maxSpeechMs | 30s | 最长录音时间，超过自动截断 |
| adaptiveNoise | true | 是否根据环境动态调整阈值 |
| noiseMargin | 8 | 在噪声基线上额外加的余量 |

### 对话完整流程

```
你开始说话
  ↓ VAD 检测到语音
录音中...（聆听中）
  ↓ 你停顿 0.9 秒
发送到后端识别（识别中...）
  ↓ STT 返回文本
发送给来福对话（思考中...）
  ↓ LLM 返回回复
合成语音并播放（来福说话中...）
  ↓ 播放结束
恢复 VAD 监听，等待下一轮
```

## 常见问题

### Q: 语音识别结果不准确？

- 尝试使用更大的模型（如 small → medium）
- 确保环境噪声不太大
- 说话时靠近麦克风

### Q: 来福说话声音有延迟？

- GPT-SoVITS 推理需要 1-3 秒，这是正常的
- 如果延迟太大，可以切换到 Edge TTS（更快但不是克隆声音）
- 预合成片段是最快的，建议为常用叫声预录音频

### Q: GPT-SoVITS 启动后一直用 Edge TTS？

- 检查控制台日志，确认模型预热是否成功
- 确认 `data/voices/laifu-clone/config.json` 中的模型文件路径正确
- 确认 GPT-SoVITS 依赖已正确安装

### Q: VAD 经常误触发 / 不灵敏？

- 环境太嘈杂：VAD 会自动调高阈值，但极端噪声下效果有限
- 可以调整 `energyThreshold`（更高 = 更不灵敏）
- `silenceDuration` 调大可减少误截断

### Q: 来福不说话了？

- 检查设置中「启用语音合成」是否打开
- 检查音量是否为 0
- 检查网络连接（Edge TTS 需要联网）

## API 参考

### POST /api/stt — 语音识别

```bash
curl -X POST http://127.0.0.1:18765/api/stt \
  -F "audio=@recording.wav" \
  -F "language=zh"
```

**响应**：
```json
{
  "text": "你好来福",
  "confidence": 0.92
}
```

### POST /api/tts — 语音合成

```bash
curl -X POST http://127.0.0.1:18765/api/tts \
  -F "text=汪！主人好！" \
  -o speech.wav
```

**响应**：音频流（WAV 或 MP3）
