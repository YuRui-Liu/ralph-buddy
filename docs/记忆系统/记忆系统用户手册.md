# 记忆系统用户手册

## 功能概述

来福拥有三层记忆结构，能记住你们的对话内容、了解你的喜好，并在未来的对话中自然地提起：

- **短期记忆**：最近 10 轮对话，保持对话连贯性
- **中期记忆**：语义向量索引，检索相关话题
- **长期记忆**：永久存储的对话历史、用户画像、重要事件

```
短期记忆（内存）       ← 最近 10 轮对话
    ↓ 超过 20 条自动压缩
中期记忆（向量数据库）  ← 语义检索，找相关话题
长期记忆（SQLite）      ← 完整历史 + 用户画像 + 手动记忆
```

---

## 工作原理

### 对话中的记忆流程

每次你和来福聊天时：

1. **检索**：从向量数据库中找到与你当前话题相关的历史对话（最多 3 条）
2. **注入**：将相关记忆 + 最近 6 轮对话 + 用户画像注入给 LLM
3. **回复**：来福基于完整上下文回复你
4. **存储**：对话同时写入短期记忆、SQLite、向量数据库

### 自动压缩与画像提取

当短期记忆积累到 20 条（10 轮对话）时，自动触发：

1. LLM 将 12 条旧对话压缩为一段摘要
2. 从对话中提取用户信息（姓名、职业、爱好等）
3. 摘要存入长期记忆，用户信息更新到画像
4. 清除已压缩的短期记忆

### 用户画像

来福会自动从对话中提取你的信息，构建用户画像：

```
关于主人你记得：
- 姓名: 小明
- 工作: 产品经理
- 喜欢的食物: 披萨
- 有一只猫叫: 花花
- 最近在学: Python
```

画像信息会自然融入来福的 system prompt，让来福在对话中自然地提起这些话题。

## 手动记忆

你可以手动告诉来福需要记住的重要事情。

### 通过记忆管理面板

1. 系统托盘右键 → 记忆管理
2. 点击「添加记忆」
3. 输入内容和重要程度（1-5）
4. 来福会记住这件事，并在相关对话中提起

### 通过 API

```bash
# 添加记忆
curl -X POST http://127.0.0.1:18765/api/memory/add \
  -H "Content-Type: application/json" \
  -d '{"content": "主人下周三有重要会议", "importance": 4}'
```

## 记忆管理

### 查看记忆

通过记忆管理面板可以：

- 查看用户画像（来福对你的了解）
- 查看手动添加的重要记忆列表
- 搜索历史记忆

### 搜索记忆

```bash
curl "http://127.0.0.1:18765/api/memory/search?query=喜欢吃什么&top_k=5"
```

### 删除单条记忆

在记忆面板中点击删除按钮，或：

```bash
curl -X DELETE http://127.0.0.1:18765/api/memory/events/3
```

### 清除所有记忆

> ⚠️ 此操作不可撤销！来福会完全忘记你们的所有对话和你的信息。

```bash
curl -X DELETE http://127.0.0.1:18765/api/memory/clear
```

## 数据存储

### 存储位置

```
python-service/data/memory/
├── memory.db          # SQLite 数据库（对话历史、用户画像、事件）
└── chromadb/          # Chroma 向量数据库（语义索引）
```

### 数据库表

| 表名 | 说明 |
|------|------|
| conversations | 完整对话历史（用户消息 + 来福回复 + 压缩摘要） |
| user_profile | 用户画像（键值对，如 "工作" → "产品经理"） |
| events | 手动添加的重要记忆（含重要程度 1-5） |
| pet_attributes | 宠物属性（顺从度、毒舌值等） |

## 配置

记忆系统开箱即用，无需额外配置。首次启动时自动创建数据库。

### 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 短期记忆容量 | 20 条 | 超过后自动压缩 |
| 压缩批次 | 12 条 | 每次压缩 12 条（6 轮） |
| RAG 检索数量 | 3 条 | 每次对话检索 3 条相关记忆 |
| 注入历史轮数 | 6 轮 | 最近 6 轮对话注入 LLM |

### Embedding 模型

向量检索使用 `all-MiniLM-L6-v2` 本地 embedding 模型。首次启动时自动下载（约 80MB）。

## 常见问题

### Q: 来福记不住之前说过的话？

- 短期记忆只保留最近 10 轮对话
- 更早的内容需要通过语义检索找到，如果话题差异太大可能检索不到
- 手动添加重要事项到记忆中，确保来福不会忘记

### Q: 用户画像信息不准确？

- 画像是 LLM 从对话中自动提取的，可能有误
- 可以通过记忆面板查看和管理画像信息
- 重新提及正确信息，来福会自动更新画像

### Q: 想让来福忘记某些事情？

- 单条手动记忆：在记忆面板中删除
- 全部重置：使用「清除所有记忆」功能
- 用户画像中的特定信息：目前需要通过 API 操作数据库

### Q: 数据安全吗？

- 所有数据存储在本地 `data/memory/` 目录
- 不会上传到任何云端
- 对话内容发送给 LLM 时遵循你配置的 LLM 提供商数据政策
